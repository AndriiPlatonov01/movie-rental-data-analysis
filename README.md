# 🎬 Movie Rental Data Analysis & SQL Automation

## 📌 Project Overview
This project demonstrates the integration of **Python and SQLite** to build a lightweight data processing engine. It simulates a movie rental business database, covering everything from schema design to advanced analytical queries.

## 🛠 Key Features & Technical Skills
- **Database Architecture:** Created relational tables (`movies`, `actors`, `customers`, `rentings`) with primary and foreign key logic.
- **Advanced SQL:** Implemented complex queries using `JOIN`, `GROUP BY`, `HAVING`, and `Aggregation functions` (AVG, COUNT) to derive business insights.
- **Python Integration:** Developed a generic wrapper function for safe SQL execution.
- **Error Handling:** Implemented robust `try-except` blocks to manage SQL exceptions and ensure program stability.
- **Data Formatting:** Built a reporting module to transform raw database tuples into human-readable reports and API-ready JSON-like dictionaries.

## 📊 Analytical Insights Solved
- Calculated average movie ratings per genre and title.
- Identified high-performing customer segments by country.
- Filtered genres with significant content volume using the `HAVING` clause.
- Automated financial reporting for movie pricing and rental performance.

## 💻 Tech Stack
- **Language:** Python 3.x
- **Database:** SQLite3
- **Concepts:** ETL (Extract, Transform, Load), Relational Databases, Defensive Programming.
